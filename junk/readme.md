## `junk.irc`
i think irc looks even more interesting in the age of language
models. basically i wanna use irc as an async repl, kinda like
midjourney but also for e.g. lisp.

## `junk.lisp`
[vau calculus](https://web.cs.wpi.edu/~jshutt/kernel.html) lisp for
generative modeling. this will probably embed other languages,
especially the condition-action stuff.

## `junk.transformers`
i had the idea to use a react-like api for neural nets.

it'd also be nice to port some models from other frameworks, and do
stuff like pull out the layer norm weights for finetuning along with
inputs and outputs, like in [Frozen Pretrained Transformers as
Universal Computation Engines
](https://ojs.aaai.org/index.php/AAAI/article/view/20729).

## `junk.turtle_graphics`
i think i can do lindenmayer-style turtle graphics, the fractal flame
algorithm, ff12 gambits, iftt rules, with a single language made of
condition-action pairs.

`translate+` `translate-` `rotate+` `rotate-` `scale+` `scale-`
analogs of the forward, rotate, etc that are standard commands for
lindenmayer presentations of turtle graphics.

`penup` `pendown` more turtle commands

`save` `restore` commands from e.g. canvas, processing, postscript

`copy` `drop` duplicate and erase turtles? probably useful

`dot` draw a dot at the current position, alternative to pendown
stuff.

`jump` nonlinear jump for fractal flame, clifford attractor, etc.

## `junk.monster_factory`
ff12-style gambits are kind of like a generative program. i wanna use
them for a text game, or a roguelite or something.
